# Infinitode 天赋树详细UI设计说明

核心设计调性：贴合Infinitode整体简约、质朴且带有科技感的视觉风格，以“清晰层级、极简交互、低冗余信息”为核心，延续游戏黑灰底色、几何化元素的设计语言，避免华丽装饰，聚焦天赋树的功能性与可读性，适配游戏整体UI体系，同时兼顾PC端与移动端操作体验，契合游戏“返璞归真、专注核心玩法”的定位。

设计核心原则：1\.  视觉统一性：严格遵循游戏原有简约科技风，色彩、图标、字体与游戏主界面、防御塔界面保持一致，不新增突兀元素；2\.  逻辑清晰性：天赋树层级、分支关系直观可见，前置解锁条件可视化，无隐藏门槛；3\.  交互轻量化：操作流程简单，反馈直观，适配塔防游戏中“快速查阅、便捷加点”的需求；4\.  信息极简性：剔除所有冗余信息，仅保留天赋节点状态、解锁关系、核心交互按钮，贴合游戏朴素且高效的设计理念。

## 一、整体布局设计（适配游戏原有UI，简约不冗余）

整体采用“侧边弹出式面板\+核心天赋树区域\+精简操作栏”的布局，不占用游戏主战斗界面过多空间，兼顾战斗时的快速切换与加点操作，贴合游戏“放置与升级”的核心玩法，同时呼应游戏主界面简约、无多余装饰的设计风格。

### 1\.1 面板基础规格

- 面板形态：侧边弹出式（默认隐藏，通过主界面指定按钮触发弹出），弹出方向为从屏幕右侧向左侧滑动，弹出后占据屏幕右侧1/3宽度（PC端）、右侧1/2宽度（移动端），不遮挡主战斗区域的核心内容（防御塔、敌人路径、资源显示）。

- 面板背景：采用游戏标志性的深灰黑色底色，无复杂纹理，仅添加轻微的低透明度网格纹理（与游戏地图网格风格一致），增强科技感的同时不破坏简约调性，边缘采用圆角处理（圆角半径与游戏其他面板保持统一），无额外边框装饰。

- 面板层级：弹出后层级高于主战斗界面，低于暂停菜单、资源面板等核心功能面板，支持点击面板外区域快速收起，适配游戏战斗时的操作节奏。

### 1\.2 区域划分（三大核心区域，逻辑清晰）

面板内部按功能划分为“顶部标题区、中间天赋树核心区、底部操作栏”三大区域，区域间无明显分隔线，仅通过间距与轻微的亮度差异区分，保持界面整洁，贴合游戏朴素的视觉风格。

- 顶部标题区：高度为面板总高度的10%，居中显示天赋树核心标题（与游戏字体风格一致），无额外装饰，左侧设置“收起按钮”（极简几何图标，与游戏其他功能按钮样式统一），点击后面板平滑收起，响应速度与游戏其他面板保持一致。

- 中间天赋树核心区：高度为面板总高度的75%，为天赋树主要展示区域，采用“线性分支结合”的布局（贴合游戏天赋多分支、多类别特点），支持上下滑动（移动端）、鼠标滚轮滚动（PC端），滚动时无卡顿，滚动条采用极简样式（低透明度，仅滚动时显示），不破坏界面整体性。

- 底部操作栏：高度为面板总高度的15%，仅保留2个核心操作按钮，按钮布局对称，样式与游戏内其他功能按钮（如防御塔升级、资源开采按钮）保持一致，无多余装饰，聚焦核心操作。

## 二、天赋树核心区域设计（层级清晰，贴合游戏风格）

核心设计：延续游戏几何化、简约化的视觉语言，天赋节点采用极简几何图形，分支路径简洁，节点状态通过颜色、亮度差异区分，不添加多余动效，仅保留核心视觉反馈，贴合游戏“质朴但不粗糙”的设计特点，同时清晰呈现天赋的分支分类（如防御塔相关、资源相关、基地相关等）。

### 2\.1 天赋层级与分支布局

- 层级划分：按游戏天赋逻辑，分为“基础层→进阶层→专精层→终极层”四个层级，层级呈纵向排列，每个层级之间保持固定间距，层级内的天赋节点横向分布，整体形成“纵向层级\+横向分支”的树状结构，清晰呈现天赋解锁的先后顺序。

- 分支划分：每个层级下按天赋类别分为若干分支（如防御塔类、资源类、基地类等），分支之间通过轻微的间距分隔，无额外分支标识，仅通过节点图标类别区分（与游戏内防御塔、资源图标风格统一），贴合游戏天赋分类逻辑。

- 布局适配：PC端天赋树节点间距适中，支持鼠标悬停查看详情；移动端节点尺寸适当缩小，间距调整为适配手指点击的大小（避免误触），分支路径适当加粗，确保视觉清晰，适配不同设备操作需求。

### 2\.2 天赋节点设计（极简几何，状态清晰）

节点设计贴合游戏几何化视觉风格，采用简洁的几何图形，无多余装饰，仅通过颜色、亮度、细微动效区分不同状态，同时呼应游戏内防御塔、资源节点的设计语言，保持视觉统一性。

- 节点形态：采用正六边形（与游戏内部分防御塔、资源节点形态呼应），尺寸统一（PC端直径18\-22px，移动端直径12\-15px），无边框，仅通过填充色与阴影区分状态，贴合游戏简约质朴的风格。

- 节点状态（4种核心状态，视觉区分清晰，无冗余）：
        

    - 未解锁状态：浅灰色填充，无阴影，无图标，灰阶显示，点击无响应，直观体现“未开放”状态，与游戏内未解锁功能的视觉风格一致。

    - 可解锁状态：低饱和度主题色填充（与游戏主色调一致，如淡蓝色、淡紫色），轻微阴影（增强立体感，不夸张），显示对应类别极简图标（几何化，与天赋类别匹配），点击可触发解锁操作，悬停时节点亮度轻微提升（反馈直观）。

    - 已解锁状态：高饱和度主题色填充（比可解锁状态颜色更深），明显阴影，图标高亮显示，节点中心添加极小的“已解锁”标识（极简对勾或圆点，不破坏节点整体简约性），点击可查看天赋详情，悬停时无额外亮度变化（避免视觉干扰）。

    - 前置未满足状态：浅灰色填充，图标半透明显示，节点添加轻微的“禁止”标识（极简斜杠，低透明度），点击时弹出极简提示（仅显示“需解锁前置天赋”，无多余文字），贴合游戏信息极简的设计原则。

- 节点图标：采用极简几何图标，与天赋类别高度匹配（如防御塔类用极简塔形、资源类用极简矿石形、基地类用极简基地轮廓），图标颜色与节点填充色一致，无渐变、无纹理，清晰易识别，贴合游戏“用形状与颜色区分功能”的设计特点。

### 2\.3 分支路径设计（简洁直观，不冗余）

- 路径形态：采用细线条（宽度1\-2px），颜色与可解锁节点填充色一致，仅连接有前置关系的节点（即上一层级节点与下一层级对应分支节点），无交叉、无多余分支，保持路径简洁，避免视觉杂乱。

- 路径状态：对应节点状态变化，未解锁节点之间的路径为浅灰色（与未解锁节点颜色一致）；可解锁/已解锁节点之间的路径为对应主题色（与节点填充色一致）；前置未满足的路径为半透明浅灰色，直观呈现天赋解锁的依赖关系。

- 交互反馈：悬停在节点上时，该节点连接的所有前置、后置路径会轻微加粗（宽度增加1px）、亮度提升，清晰提示当前节点的关联节点，无多余动效，贴合游戏简约的交互风格。

### 2\.4 天赋详情弹窗（极简信息，快速查阅）

弹窗设计延续游戏简约风格，仅显示核心信息，无多余装饰，响应快速，不遮挡天赋树核心区域，适配“快速查阅、不影响操作”的需求，贴合游戏高效的玩法节奏。

- 弹窗触发：点击可解锁/已解锁节点时，在节点右侧弹出弹窗（不遮挡节点本身及关联路径），弹窗大小固定，仅能容纳核心信息，无拉伸、缩放效果。

- 弹窗样式：深灰色底色（与天赋树面板底色一致），无边框，轻微阴影，圆角处理（与面板圆角半径一致），弹窗内无多余装饰，仅显示核心内容。

- 弹窗内容：仅包含2类核心信息，排版简洁，字体与游戏内其他文本样式一致（清晰、简约，无衬线字体），无多余描述，具体如下：
        

    - 第一行：天赋类别标识（极简图标\+简短类别名称，如防御塔类、资源类），字体略大，颜色与节点填充色一致。

    - 第二行：天赋核心效果描述（简洁明了，无冗余，不包含任何数值），字体适中，浅白色（与面板底色形成清晰对比，易阅读）。

- 弹窗关闭：点击弹窗外区域、再次点击对应节点，或切换至其他节点时，弹窗自动关闭，关闭时无多余动画，响应迅速，不影响操作节奏。

## 三、操作栏设计（核心功能，极简交互）

操作栏聚焦“加点、重置”两大核心功能，按钮设计贴合游戏原有样式，无多余装饰，操作逻辑简单，适配游戏“快速加点、灵活调整”的需求，同时保持界面简约整洁。

- 按钮布局：两个核心按钮对称分布在操作栏两侧，中间预留空白区域（不添加任何元素），按钮尺寸统一（PC端宽度60\-80px，移动端宽度50\-70px），高度与操作栏高度一致，圆角处理（与游戏其他按钮圆角一致）。

- 按钮样式：
        

    - 按钮底色：深灰色（与面板底色一致），hover时底色轻微变亮（无渐变，仅亮度调整），点击时底色短暂加深（反馈直观，无多余动效）。

    - 按钮图标：采用极简几何图标，与按钮功能匹配（如加点按钮用“\+”，重置按钮用“循环箭头”），图标颜色与天赋节点主题色一致，无多余装饰，清晰易识别。

    - 按钮文本：仅在按钮下方添加极简文本（与图标功能对应，字体极小，浅白色），无多余描述，贴合游戏简约风格，PC端可显示，移动端可隐藏（仅保留图标，节省空间）。

- 操作逻辑：
        

    - 加点按钮：仅在存在“可解锁”节点时激活（底色变亮，图标高亮），点击后，若当前有可分配点数，自动为“当前选中节点”解锁（若未选中节点，无响应）；若无可分配点数，按钮灰化，点击无响应，无额外提示（贴合游戏极简信息原则）。

    - 重置按钮：始终处于激活状态（除非无已解锁节点），点击后弹出极简确认弹窗（仅显示“确认重置所有天赋？”\+“确认/取消”两个选项），确认后所有已解锁节点恢复为“可解锁”状态，点数全部返还，弹窗样式与天赋详情弹窗保持一致，无多余装饰。

## 四、视觉细节与适配设计（贴合游戏风格，兼顾体验）

### 4\.1 视觉细节（简约加分，不冗余）

- 色彩搭配：严格遵循游戏原有色彩体系，以深灰黑色为底色，低饱和度主题色（淡蓝、淡紫等）为点缀，无鲜艳刺眼的颜色，所有元素颜色统一，贴合游戏“朴素、简约”的视觉特点，同时确保不同状态的节点、路径区分清晰。

- 字体规范：采用游戏内统一的无衬线字体，字体大小层级清晰（标题\&gt;类别名称\&gt;效果描述\&gt;按钮文本），颜色为浅白色（与深灰黑色底色形成清晰对比），无加粗、斜体等多余样式（除标题外），确保阅读清晰，贴合游戏文本简约的设计风格。

- 动效设计：仅保留3种核心极简动效，无多余华丽动画，贴合游戏整体动效风格：
        

    - 面板弹出/收起：平滑滑动（时长0\.2\-0\.3秒），无卡顿、无弹跳，与游戏其他面板动效保持一致。

    - 节点解锁：解锁时节点有轻微“点亮”效果（亮度从低到高渐变，时长0\.1\-0\.2秒），无其他多余动效，强化解锁反馈。

    - 按钮交互：hover时亮度轻微变化，点击时短暂底色加深（时长0\.1秒），反馈直观，不影响操作节奏。

- 细节适配：天赋树核心区边缘添加轻微的渐变阴影（从深灰到透明），避免与面板底色生硬衔接；节点之间的间距统一，确保布局整齐，无杂乱感；所有可交互元素（节点、按钮）均添加轻微的点击区域放大（不改变视觉尺寸，仅扩大可点击范围），避免误触。

### 4\.2 多设备适配（兼顾PC端与移动端，不改变核心设计）

- PC端适配：
        

    - 面板宽度：占据屏幕右侧1/3，天赋树节点尺寸适中（直径18\-22px），间距合理，支持鼠标滚轮滚动、鼠标悬停查看详情。

    - 操作适配：支持鼠标点击节点、按钮，悬停节点时路径高亮，详情弹窗快速弹出，适配PC端精准操作的特点。

    - 额外优化：天赋树核心区可通过鼠标拖拽移动（方便查看不同区域的天赋节点），拖拽时无卡顿，松手后自动定位，贴合PC端操作习惯。

- 移动端适配：
        

    - 面板宽度：占据屏幕右侧1/2，天赋树节点尺寸缩小（直径12\-15px），间距放大（适配手指点击，避免误触），分支路径适当加粗，确保视觉清晰。

    - 操作适配：支持手指点击、滑动（上下滑动查看天赋树），点击节点弹出详情弹窗，按钮尺寸放大（适配手指点击），隐藏按钮文本（仅保留图标，节省空间）。

    - 额外优化：添加“快速回到顶部”按钮（极简图标，位于天赋树核心区右下角，仅滑动时显示），点击可快速回到基础层，提升操作效率；弹窗尺寸适当缩小，避免遮挡过多区域。

### 4\.3 与游戏原有UI的统一性适配

- 风格统一：天赋树面板、节点、按钮的设计风格，与游戏主界面、防御塔升级面板、资源面板保持一致，均采用深灰黑色底色、极简几何元素、低饱和度点缀色，不新增突兀元素，贴合游戏整体“简约、质朴、科技感”的调性。

- 元素统一：节点图标、按钮图标与游戏内防御塔、资源、功能按钮的图标风格一致，均为极简几何形，无多余装饰；字体、圆角、阴影等细节，与游戏其他UI元素保持统一，确保整体视觉连贯。

- 交互统一：面板弹出/收起、按钮点击、弹窗关闭等交互逻辑，与游戏其他面板（如防御塔详情面板、设置面板）保持一致，降低玩家学习成本，贴合游戏操作习惯，同时呼应游戏“人性化按键、高效操作”的特点。

## 五、异常状态与提示设计（极简提示，不干扰操作）

提示设计遵循“极简、必要”原则，仅在关键异常状态下显示提示，无多余文字、无夸张弹窗，贴合游戏信息极简的设计风格，不干扰玩家战斗与加点操作。

- 无可分配点数：点击加点按钮时，无任何弹窗提示，仅按钮保持灰化状态（与未解锁节点颜色一致），直观提示“无可分配点数”，避免多余提示干扰操作。

- 前置未满足：点击前置未满足的节点时，弹出极简提示（浅灰色小弹窗，仅显示“需解锁前置天赋”，无多余文字），弹窗显示1秒后自动关闭，不遮挡核心区域，反馈清晰且不干扰操作。

- 重置确认：点击重置按钮时，弹出极简确认弹窗（深灰色底色，仅显示确认、取消选项，无多余装饰），确认后执行重置操作，取消后关闭弹窗，贴合游戏极简交互原则。

# 月蚀幸存者

本地网页 roguelite 生存游戏。当前版本包含章节故事模式、2-5 人房间制联机、本机账号、长期养成、排行榜和本地打包的 CC0 像素素材。

## 运行

```powershell
npm start
```

打开：

```text
http://localhost:5173
```

账号、存档、排行榜保存在 `data/store.json`。

## 联机配置

### 本机自己玩

```text
http://localhost:5173
```

### 同一个 WiFi / 局域网

先查本机局域网 IP：

```powershell
Get-NetIPAddress -AddressFamily IPv4
```

选择 `WLAN` 或正在联网的网卡地址，例如：

```text
http://10.102.118.101:5173
```

把这个地址发给同一 WiFi 下的朋友。你创建房间后，再把房间码发给他们。

### cpolar 内网穿透

先启动游戏：

```powershell
npm start
```

另开一个终端窗口启动 cpolar：

```powershell
cpolar http 5173
```

cpolar 会显示公网地址，通常类似：

```text
https://xxxxx.cpolar.top
```

把这个公网地址和游戏房间码发给朋友。朋友打开公网地址后，注册/登录，进入“联机”，输入房间码即可加入。

### 联机流程

1. 房主打开游戏并登录。
2. 进入“联机”，创建房间。
3. 把访问地址和房间码发给朋友。
4. 朋友打开地址，注册/登录。
5. 朋友进入“联机”，输入房间码。
6. 所有人准备后，房主开始。

### 常见问题

- 朋友打不开页面：确认 `npm start` 和 `cpolar http 5173` 都还在运行。
- 能打开页面但进不了房间：确认房间码没输错，房间还没结束，房间人数没有超过 5 人。
- 换了 cpolar 地址：免费临时地址可能会变，需要把新地址重新发给朋友。
- 数据在哪里：账号、存档、排行榜都在 `data/store.json`。

## 项目约定

- 所有运行、部署、联机、端口、穿透、账号数据位置等配置相关内容，都要同步写进本 README。
- 改动联机端口时，同时更新 `server.js` 默认端口和 README 示例。
- 新增素材时，同时更新 `assets/ATTRIBUTION.md`。

## 内容

- 50 个章节：10 个主题区域 × 5 幕，后续可继续扩展生成模板
- 用户名密码登录，本机保存账号数据，密码使用 `crypto.scrypt` 哈希
- 月尘、长期星盘、角色/遗物解锁、故事进度、好友、图鉴进度和排行榜
- 长期星盘包含攻击、生存、机动、资源、协作、深层 6 个分支，部分节点随章节推进解锁，等级上限和成本曲线适合长期游玩
- 2-5 人联机房间：创建、加入、准备、开始、掉线托管、倒地救援、全员结算
- 房间大厅支持复制房间码、房主踢人、好友邀请和最多 3 个房间词条
- 好友系统使用本机账号用户名添加，不需要邮箱或手机号
- 排行榜分今日/历史、得分/击杀 4 种视图
- 图鉴包含词条解释、怪物、Boss、局内升级和房间词条说明
- 局内技能随章节推进解锁；创建/加入房间前会展示本局技能池和可触发的进化联动
- 好友添加采用申请/同意流程，被添加者确认后才会成为好友
- 服务端模拟房间战斗状态，客户端提交移动和升级选择
- 角色、音乐、音效和 UI 素材来自 Ninja Adventure 与 Kenney CC0 包，来源见 `assets/ATTRIBUTION.md`

## 深度拓展记录

- 章节不再只是数值递增：每关会带 2-3 个词缀，例如精英巡猎、月毒雾潮、虚空落星、枯竭补给、古灯余辉、远咒回廊等。
- 每章带不同目标模板：猎杀推进、仪式压制、坚守远征、遗物搜寻、斩首行动，后续扩章节时优先扩 `makeChapterAffixes` 和 `makeChapterObjective`。
- 局内升级从 6 个扩展到 12 个：伤害、速度、冷却、生命、多重射击、范围阵、穿透、暴击、磁吸、射程、护盾、恢复。
- 新增战斗差异：关卡词缀会影响刷怪密度、精英波次、远程敌人、地面危险区、补给频率和敌人速度。
- 长期养成与局内流派联动：长期星盘偏向开局底子，局内升级偏向当局构筑，两者不要设计成完全重复的数值堆叠。

## 联机房间机制

- 房主创建房间时可以选择 0-3 个房间词条，词条会影响全队生命、经验、敌人强度、补给、宝箱、得分或月尘。
- 创建房间页展示一组随机词条，点击“刷新”会重新生成候选词条；候选词条最多选择 3 个进入房间。
- 房间码可在大厅一键复制；通过 cpolar 给朋友公网地址，再把房间码发给朋友即可加入。
- 房主可以在开局前踢出成员；开局后不允许踢人，掉线玩家会进入托管。
- 好友在线时可以从房间大厅点“邀请”，对方会收到房间码提示，也可以手动输入房间码加入。

## 技能与进化

- `upgradeDefs` 定义局内技能，包括解锁章节、最高等级、说明、图标、颜色和提升属性。
- `evolutionDefs` 定义满级联动，例如伤害 + 穿透满级后进化为“日冕长枪”。
- 章节 ID 会过滤技能池，前期只出现基础技能，后期逐步加入月环刃、坠星新星、血月诅咒、守夜傀儡等高级技能。
- 升级弹窗会显示当前等级、最高等级、提升属性和可能触发的满级联动。

## 界面约定

- 主菜单采用侧边导航 + 固定内容区，避免切换页面时版块跳动。
- 图鉴采用左侧分类/条目、右侧详情的分栏布局，避免大段文字一次性铺满。
- 图鉴也采用独立全屏界面：点击“图鉴”会离开主菜单，进入幻想手册式界面；左侧是分类书签，中间是条目列表，右侧是大图、属性条和作战提示，交互参考 Kingdom Rush 的图鉴结构但不逐像素复制原界面。
- 联机创建区比加入区更大，因为创建房间需要展示章节、随机词条和技能池；加入房间只保留房间码入口。
- 长期养成参考 Infinitode 的研究界面方式：点击“养成”会离开主菜单，进入独立全屏研究地图；页面包含顶部资源栏、左侧分支筛选、中央可滚动研究地图、小型节点、网格背景和发光连线。点击节点再进入独立研究详情页，详情页负责升级、前置、效果说明。服务端也会校验前置天赋，不能只靠前端显示限制。
